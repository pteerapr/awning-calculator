# คู่มือแก้ไขราคาเครื่องคำนวณกันสาด (สำหรับเจ้าหน้าที่)

คู่มือนี้สำหรับ **เจ้าหน้าที่ธุรการ** ที่ต้องอัปเดตราคาในเครื่องคำนวณ (ไฟล์ `.html`) เมื่อมีราคาชุดใหม่จากโรงงาน
ไม่ต้องเขียนโปรแกรมเป็นก็ทำได้ ขอแค่ทำตามขั้นตอนอย่างระมัดระวัง

> **สรุปสั้น ๆ ก่อนเริ่ม**
> 1. หาไฟล์ที่ถูกต้อง → 2. แก้เฉพาะ "ตัวเลข" → 3. เพิ่มเลขเวอร์ชันใน `sw.js` → 4. ทดสอบ → 5. ส่งขึ้น GitHub

---

## ⚠️ กฎเหล็ก 6 ข้อ (อ่านก่อนแก้ทุกครั้ง)

1. **แก้ได้เฉพาะตัวเลข** เท่านั้น อย่าลบเครื่องหมาย `,` `[` `]` `{` `}` `"` `:` หรือ `//`
2. **ห้ามใส่ลูกน้ำในตัวเลข** ในโค้ด — พิมพ์ `58070` ไม่ใช่ `58,070`
3. **แก้ทีละไฟล์ ทีละรุ่น** อย่าแก้หลายอย่างพร้อมกัน จะได้ตรวจง่ายถ้าผิด
4. **คัดลอกไฟล์เก็บสำรองก่อนแก้เสมอ** (Copy วางไว้อีกที่ก่อน)
5. **ทุกครั้งที่แก้ ต้องเพิ่มเลขเวอร์ชันใน `sw.js`** ไม่งั้นลูกค้าจะยังเห็นราคาเก่า (ดูหัวข้อที่ 5)
6. **ทดสอบในเบราว์เซอร์ก่อน deploy ขึ้นเว็บจริงเสมอ** อย่าเผยแพร่ทั้งที่ยังไม่ได้ลอง

## 1. ทำความรู้จักไฟล์ต่าง ๆ

| ไฟล์ | คืออะไร | เปิดด้วย |
|---|---|---|
| `terra-ma3.html`, `seatel-st2000.html`, `chill-d1.html` ฯลฯ | เครื่องคำนวณ 1 ไฟล์ = 1 รุ่นสินค้า | เบราว์เซอร์ (ดู) / Notepad หรือ VS Code (แก้) |

## 2. ราคามี 3 แบบ อยู่คนละที่

ก่อนแก้ ต้องรู้ว่าราคาที่จะแก้เป็น **แบบไหน** เพราะอยู่คนละที่

### แบบ A — ราคาตามขนาด (อยู่ในไฟล์ของรุ่นนั้น ๆ)
ราคาที่เปลี่ยนตาม **ความกว้าง × ระยะยื่น** เช่น ตารางราคากันสาด Terra MA3
→ อยู่ใน **ไฟล์ HTML ของรุ่นนั้น** (เช่น `terra-ma3.html`)

### แบบ B — ราคาอุปกรณ์เสริม (อยู่ใน `prices.js`)
รีโมท 1 ช่อง, รีโมท 15 ช่อง, เซ็นเซอร์ลม-แดด, อุปกรณ์ Somfy ฯลฯ
→ อยู่ใน **`prices.js` ที่เดียว** แก้ที่นี่ครั้งเดียว ทุกรุ่นที่ใช้ร่วมกันจะเปลี่ยนตามอัตโนมัติ

### แบบ C — ราคาตัวมอเตอร์ (อยู่ในไฟล์ของรุ่นนั้น ๆ)
ราคามอเตอร์แต่ละรุ่น มักอยู่ในบรรทัด `<option ...>` ภายในไฟล์ HTML ของรุ่นนั้น

## 3. วิธีแก้ "ราคาตามขนาด" (แบบ A)

**ตัวอย่าง:** ราคากันสาด Terra MA3 ผ้า Sunbrella ระบบมือหมุน ปรับขึ้น

### 3.1 เปิดไฟล์
เปิด `terra-ma3.html` ด้วย VS Code

### 3.2 ค้นหาตารางที่ถูกต้อง
กด `Ctrl + F` แล้วพิมพ์ชื่อตาราง ตารางราคาจะตั้งชื่อแบบนี้:

- `PRICE_MANUAL_...` = ระบบ **มือหมุน**
- `PRICE_MOTOR_...` = ระบบ **มอเตอร์**
- ท้ายชื่อคือชนิดผ้า เช่น `_ACRYLIC` `_SOLAR` `_SUNBRELLA` `_NOVA`

เช่น อยากแก้ผ้า Sunbrella มือหมุน → ค้นหา `PRICE_MANUAL_SUNBRELLA`

### 3.3 เข้าใจโครงสร้างตาราง
ตารางหน้าตาแบบนี้ (ตัวเลขในวงเล็บ `//` คือคำอธิบาย ไม่ต้องแก้):

```js
const PRICE_MANUAL_SUNBRELLA = {
  "1.50": [        // ← ระยะยื่น 1.50 เมตร
    25350, // 2.00–2.50   ← ความกว้าง (ช่องแรก)
    27780, // 2.51–3.00
    30270, // 3.01–3.50
    ...
  ],
  "2.00": [        // ← ระยะยื่น 2.00 เมตร
    null,  // 2.00–2.50 ไม่มี   ← null = ช่องที่ขายไม่ได้ (ในตารางเป็น x)
    31130, // 2.51–3.00
    ...
```

**อ่านตารางยังไง:**
- บรรทัด `"1.50":` `"2.00":` ฯลฯ = **ระยะยื่น (เมตร)**
- ตัวเลขแต่ละบรรทัดในวงเล็บ `[ ]` = **ช่วงความกว้าง** เรียงจากแคบ (2.00–2.50) ไปกว้าง (7.51–8.00)
- คำอธิบายหลัง `//` บอกว่าเลขนั้นคือช่วงความกว้างไหน

### 3.4 แก้ตัวเลข
หาช่องที่ต้องการ แล้วพิมพ์ทับเฉพาะตัวเลข **อย่าลืมเครื่องหมาย `,` ท้ายบรรทัดไว้เหมือนเดิม**

```js
    25350, // 2.00–2.50      ← เดิม
    26100, // 2.00–2.50      ← แก้เป็นราคาใหม่ (คง , ไว้)
```

- ในตาราง PDF ช่องไหนเป็น **x** → ในโค้ดใช้คำว่า **`null`** (อย่าใส่ 0)
- **ห้ามใส่ลูกน้ำในตัวเลข** → `26100` ✅ ไม่ใช่ `26,100` ❌

### 3.5 บันทึกไฟล์
กด `Ctrl + S`

## 4. วิธีแก้ "ราคาอุปกรณ์เสริม" (แบบ B — `prices.js`)

**ตัวอย่าง:** รีโมท 1 ช่อง ขึ้นจาก 950 เป็น 990 บาท

### 4.1 เปิด `prices.js`

### 4.2 หาบรรทัดที่ต้องแก้
ราคาอยู่เป็นรายการชัดเจน เช่น:

```js
  strongUp: {
    remote1:      950,   // Transmitter 1 Channel / รีโมท 1 ช่อง
    remote15:    1800,   // Transmitter 15 Channel / รีโมท 15 ช่อง
    windSun:     6600,   // Wind-Sun Sensor / เซ็นเซอร์ลม-แดด
    ...
```

### 4.3 แก้เฉพาะตัวเลข (คง `,` ไว้)

```js
    remote1:      990,   // Transmitter 1 Channel / รีโมท 1 ช่อง
```

> **ข้อดี:** แก้ `prices.js` ครั้งเดียว ทั้ง "ราคาที่คำนวณ" และ "ป้ายราคาที่แสดงในหน้าจอ" จะเปลี่ยนตามอัตโนมัติทุกรุ่นที่ใช้ร่วมกัน ไม่ต้องไล่แก้ทีละไฟล์

### 4.4 บันทึกไฟล์ `Ctrl + S`

```js
const CACHE_NAME = 'sangthong-awning-v4';
```

## 6. ทดสอบก่อนเผยแพร่ (ก่อน deploy ขึ้นเว็บจริง)

1. เปิดไฟล์เครื่องคำนวณที่แก้ (เช่น `terra-ma3.html`) ด้วยเบราว์เซอร์ (ดับเบิลคลิก)
2. ลองเลือกขนาด/ผ้า/อุปกรณ์ ที่ตรงกับราคาที่เพิ่งแก้
3. เช็กว่า **ราคาที่แสดงตรงกับราคาใหม่** ที่ตั้งใจไว้
4. เช็กว่าไม่มีคำว่า `NaN`, `undefined`, หรือช่องราคาว่างเปล่า
5. ถ้าเบราว์เซอร์ยังโชว์ราคาเก่า ให้กด `Ctrl + Shift + R` (โหลดใหม่แบบล้างแคช)

> ถ้าหน้าจอเพี้ยน/ราคาไม่ขึ้นเลย แปลว่าอาจพิมพ์ผิด (ลบ `,` หรือ `]` ไป) → ดูหัวข้อที่ 8

## 7. เผยแพร่ขึ้นเว็บจริง (Cloudflare Pages)

> เว็บใช้งานจริงอยู่ที่ **https://awning-calculator.pages.dev** โฮสต์บน **Cloudflare Pages**
> ⚠️ การแก้ไฟล์ในเครื่องหรือบน GitHub เฉย ๆ **จะไม่อัปเดตเว็บจริง** — ต้อง "deploy" ขึ้น Cloudflare ตามขั้นตอนนี้

### 7.1 เปิด Terminal ในโฟลเดอร์โปรเจกต์
- ใน **VS Code**: เมนู **Terminal → New Terminal**
- (หรือเปิด **PowerShell** แล้ว `cd` เข้าโฟลเดอร์โปรเจกต์)

### 7.2 สั่ง deploy ด้วยคำสั่งเดียว
คัดลอกคำสั่งนี้ไปวางแล้วกด Enter:

```bash
npx wrangler pages deploy . --project-name awning-calculator --branch main --commit-dirty=true
```

- รอสักครู่ จะขึ้นข้อความ **✨ Deployment complete!** พร้อมลิงก์
- เว็บจริง **https://awning-calculator.pages.dev** จะอัปเดตภายในไม่กี่วินาที

### 7.3 ครั้งแรกสุดต้อง login ก่อน (ทำครั้งเดียว)
ถ้าขึ้นว่ายังไม่ได้เข้าสู่ระบบ ให้พิมพ์:
```bash
npx wrangler login
```
เบราว์เซอร์จะเปิดให้กด **Allow** (บัญชี Cloudflare: pteerapr@gmail.com) แล้วค่อยสั่ง deploy (7.2) ซ้ำ

### 7.4 เห็นราคาเก่าอยู่? → ล้างแคช
เว็บนี้เก็บข้อมูลไว้ในเครื่อง (PWA) หลัง deploy ให้กด **`Ctrl + Shift + R`**
ถ้ายังไม่หาย: กด `F12` → แท็บ **Application** → **Storage** → **Clear site data** → รีโหลด

### 7.5 (ทางเลือก) เก็บสำเนาโค้ดไว้บน GitHub
ไม่บังคับ แต่แนะนำให้เก็บประวัติโค้ดไว้เผื่อย้อนกลับ — ใช้ **GitHub Desktop**:
พิมพ์คำอธิบายในช่อง **Summary** → **Commit to main** → **Push origin**
> GitHub เก็บแค่ "ประวัติโค้ด" — หน้าเว็บที่ลูกค้าเห็นจริงอัปเดตจากขั้นตอน deploy Cloudflare (7.2) เท่านั้น

## 8. แก้ปัญหาที่พบบ่อย

| อาการ | สาเหตุที่พบบ่อย | วิธีแก้ |
|---|---|---|
| ราคาไม่ขึ้น / หน้าเพี้ยน | ลบเครื่องหมาย `,` `]` `}` หรือ `"` ไปโดยไม่ตั้งใจ | เอาไฟล์สำรองที่ copy ไว้มาเทียบ แล้วใส่กลับ |
| ราคาเพี้ยนมาก (เช่น เป็นหลักสิบ) | ใส่ลูกน้ำในตัวเลข เช่น `26,100` | ลบลูกน้ำออก เหลือ `26100` |
| แก้แล้วแต่ยังเห็นราคาเก่า | ลืมเพิ่มเลข v ใน `sw.js` | เพิ่มเลข v (หัวข้อ 5) แล้ว deploy ใหม่ (หัวข้อ 7) |
| เบราว์เซอร์ตัวเองยังเห็นเก่า | แคชในเครื่อง | กด `Ctrl + Shift + R` |
| ช่องที่ควร "ไม่มี" กลับมีราคา 0 | ใส่ `0` แทน `null` | เปลี่ยนเป็น `null` |

> **ถ้าแก้แล้วพัง แก้กลับไม่ได้:** อย่าเพิ่ง deploy ขึ้นเว็บจริง ให้เอาไฟล์สำรองที่ copy ไว้ก่อนแก้มาวางทับ แล้วเริ่มใหม่
